# API Reference

Base prefix in backend route handlers: `/api/*`.

> [!NOTE]
> This section is generated from backend `route.ts` handlers and frontend API clients. Where payload details are not explicit in client types, fields are marked **Needs verification**.

## Auth endpoints

| Method | Path | Purpose | Auth | Frontend usage |
| --- | --- | --- | --- | --- |
| POST | `/api/auth/login` | Authenticate and issue access+refresh tokens | Public | `auth-api.login` |
| POST | `/api/auth/refresh` | Rotate refresh token and issue new access token | Refresh cookie | `auth-api.refresh`, Axios interceptor |
| POST | `/api/auth/logout` | Revoke refresh token and clear cookie | Bearer | `auth-api.logout` |
| GET | `/api/auth/me` | Current authenticated user profile | Bearer | `auth-api.getMe` |
| POST | `/api/auth/forgotPassword` | Generate reset flow email | Public | `auth-api.forgotPassword` |
| PATCH | `/api/auth/changePwd` | Validate or complete reset/activation token flow | Public token-based | Not directly typed in frontend app |

## User endpoints

| Method | Path | Purpose | Auth |
| --- | --- | --- | --- |
| POST | `/api/users/create` | Create user and trigger activation flow | Admin |
| GET | `/api/users/list` | Paginated user listing | Admin |
| GET | `/api/users/getById?id=` | User detail by id | Admin |
| PATCH | `/api/users/update` | Update user | Admin |
| DELETE | `/api/users/remove?id=` | Remove user (constraints apply) | Admin |
| PATCH | `/api/users/changeMyPwd` | Change own password | Bearer |
