# Architecture

PakAG is a **pnpm monorepo** with three packages:

```
ERRONKA2025_JS/
  backend-js/    -> @repo/backend-js  (REST API)
  frontend-app/  -> @repo/frontend-app (Next.js App)
  docs/          -> @repo/docs        (This documentation)
  schema.sql     -> Database schema
```

## Data flow

```
Client (frontend-app)
  -> REST API (backend-js /api/*)
    -> Service (logic)
      -> Repository (SQL / mysql2)
        -> MySQL (pakAG_db)
```

## Communication between layers

- **route.ts**: receives Request, validates with DTO, calls the service, returns Response
- **service**: pure logic, no HTTP or SQL dependencies
- **repository**: prepared statements only, returns domain types

## Authentication

JWT signed with `HS256`. The token is sent in the `Authorization: Bearer <token>` header.  
The `src/lib/jwt.ts` middleware verifies and extracts the payload on every protected request.

## Emails

Resend with custom HTML templates (no external template library). Templates live in  
`src/app/lib/email/templates/` and include responsive design with package status tracking.
