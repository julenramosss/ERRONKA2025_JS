# AI dokumentazio fluxua

## Rolak eta lankidetza-eredua

### Codex (dokumentazioaren eraikitzaile nagusia)

- Biltegia eskaneatu eta iturri-fitxategien aurka egiaztapen teknikoak egiten ditu.
- `/docs`-ko dokumentazio-arkitektura eraiki eta mantentzen du.
- Orrialde tekniko oso eta egituratutakoak sortzen ditu.

### Copilot (itzulpena eta atal txikiak)

- Iturri ingelesa espainolera eta euskarara itzultzen du.
- Zabalpen txikiak eta formatu-fintzeak kudeatzen ditu.
- Ez luke egiaztapenik gabe oinarrizko aldarrikapen teknikoak aldatu behar.

### Claude (berrikuspen eta argitalpen-kalitate kontrola)

- Zuzentasuna, koherentzia eta irakurgarritasuna berrikusi.
- Hutsuneak, kontraesanak eta erreferentzia falta seinalatzen ditu.
- Mantengarritasun-hobetze estrukturalak gomendatzen ditu.

## Negoziaezineko arauak

- Inongo AI agenteak ez du sistema-portaera asmatu behar.
- Ziurgabeak diren arlo guztiak **Egiaztapena behar du** gisa modu esplizituan markatu behar dira.
- Aldarrikapen tekniko bakoitza uneko biltegiaren kodean trazagarria izan behar da.

## Iradokitako eskualdatze-sekuentzia

1. Codex-ek ingelese iturri-orriak idatzi/eguneratzen ditu.
2. Claude-k berrikusi eta zuzenketak eskatzen ditu.
3. Copilot-ek hiztegiak/edukia itzultzen ditu iturria egonkor denean.
4. Codex-ek azken koherentzia-pasa egiten du eta argitaratzen du.
